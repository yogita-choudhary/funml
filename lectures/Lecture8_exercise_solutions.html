Lecture 8: Polynomial Regression (Gradient Descent Iterations)

1.1 Question

Context (from lecture): In Gradient Descent training,

  • Batch GD: 1 iteration per epoch.

  • Stochastic GD: \(N\) iterations per epoch.

  • Mini-batch GD: \(N/b\) iterations per epoch.

Given: \[N = 1{,}000{,}000, \qquad P = 100, \qquad b = 10{,}000\]

Tasks:

  1. (Iterations per epoch) Compute the number of iterations in one epoch for:

    1. Batch GD

    2. Stochastic GD

    3. Mini-batch GD (batch size \(b=10{,}000\))

  2. (Memory constraint) Suppose you can process at most \(50{,}000\) data points at a time. Which methods can be used without exceeding this memory limit?

    1. Only Stochastic GD

    2. Stochastic GD and Mini-batch GD with \(b \le 50{,}000\)

    3. Batch GD and Mini-batch GD with \(b = 50{,}000\)

    4. All methods

  3. (Multiple epochs) If you perform 10 epochs of Mini-batch GD with batch size \(b=20{,}000\), how many total iterations will you run?

Answers should be exact integers.

1.2 Solutions

Given: \[N = 1{,}000{,}000, \qquad b = 10{,}000\]

Solutions:

  1. Iterations per epoch

    1. Batch GD: One update uses the entire dataset \(\Rightarrow \boxed{1 \text{ iteration}}\).

    2. Stochastic GD: One update per sample \(\Rightarrow \boxed{N = 1{,}000{,}000 \text{ iterations}}\).

    3. Mini-batch GD: One update per batch \(\Rightarrow \boxed{N/b = 1{,}000{,}000/10{,}000 = 100 \text{ iterations}}\).

  2. Memory constraint Batch GD must process all \(N=1{,}000{,}000\) points at once (too large). Stochastic uses 1 point. Mini-batch works if \(b \le 50{,}000\). \[\boxed{\text{Correct choice: (b)}}\]

  3. 10 epochs of Mini-batch GD with \(b=20{,}000\) \[\text{Batches per epoch} = N/b = 1{,}000{,}000/20{,}000 = 50\] \[\text{Total iterations} = 10 \times 50 = \boxed{500}\]

Final Answers: \[\boxed{(1)\ a=1,\ b=1{,}000{,}000,\ c=100}\] \[\boxed{(2)\ \text{choice (b)}}\] \[\boxed{(3)\ 500}\]
