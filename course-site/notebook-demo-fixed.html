Welcome to this demo page showcasing various ways to share interactive computational notebooks. Whether you're working with Python, Julia, R, or Wolfram Language, there are multiple platforms that allow you to share your work in an accessible format.

Click the buttons below to open example notebooks in different platforms. Each platform has unique strengths for different use cases.

üìä Google Colab Notebook

Google Colab is ideal for Python-based machine learning and data science workflows. It provides free GPU/TPU access and integrates seamlessly with Google Drive.

Features:
  • Free GPU/TPU access
  • Pre-installed popular libraries (TensorFlow, PyTorch, etc.)
  • Google Drive integration
  • Collaborative editing

üìì Jupyter Notebook (nbviewer)

nbviewer provides a clean, static rendering of Jupyter notebooks. Perfect for sharing analyses, tutorials, or documentation without requiring users to run code.

Features:
  • Fast, static rendering
  • No execution environment needed
  • Clean, readable output
  • Works with any public .ipynb file
üìñ View Example Notebook üìÇ View Your Notebook
How to use with your notebooks:
https://nbviewer.org/github/YOUR-USERNAME/YOUR-REPO/blob/main/notebook.ipynb
üí° Best for: Documentation, tutorials, blog posts, or any content where readers don't need to modify/execute the code.

üî¨ Interactive Binder Environment

Binder creates a fully interactive computing environment in the cloud. Users can modify code, install packages, and experiment freely. Supports Python, R, Julia, and more.

Features:
  • Full interactive Jupyter environment
  • No login required
  • Supports multiple languages (Python, R, Julia)
  • Custom dependencies via requirements.txt or environment.yml
üöÄ Launch Example in Binder üìÇ Launch Your Notebook
How to use with your notebooks:
https://mybinder.org/v2/gh/YOUR-USERNAME/YOUR-REPO/main?filepath=notebook.ipynb

‚è±Ô∏è Note: First launch takes 30-60 seconds to build the environment. Subsequent launches are faster.

üßÆ Wolfram Cloud Notebook

Wolfram notebooks bring symbolic computation, visualization, and the Wolfram Language to the web. Great for mathematical modeling, symbolic math, and computational essays.

Features:
  • Symbolic computation and CAS
  • Advanced visualization
  • Interactive manipulations
  • Knowledge-based computing with Wolfram|Alpha integration
üßÆ View Example Computation ‚òÅÔ∏è Wolfram Cloud
To share your Wolfram notebooks:
  1. Create your notebook in Mathematica or Wolfram Cloud
  2. Publish it to Wolfram Cloud (File ‚Üí Publish to Cloud)
  3. Get the cloud object URL (looks like: https://www.wolframcloud.com/obj/username/notebook-name)
  4. Share the URL or embed it in your page

üìÅ GitHub Notebook Viewer

GitHub automatically renders .ipynb files when you view them in a repository. This is the simplest way to share read-only notebooks.

Features:
  • No setup required
  • Automatic rendering on GitHub
  • Version control built-in
  • Easy to link directly to notebooks

üí° Pro tip: Just commit your .ipynb files to GitHub and link to them. GitHub will render them automatically!

üíæ Alternative: Export as HTML

For complete control over how your notebook appears, convert it to static HTML and host it directly on GitHub Pages.

How to export notebooks to HTML:

Jupyter Notebook:
jupyter nbconvert --to html notebook.ipynb

Or from within Jupyter:
File ‚Üí Download as ‚Üí HTML (.html)

Then upload the HTML file to your GitHub Pages repository and link to it directly!
üí° Advantages: No external dependencies, fastest loading, works offline, complete styling control, no iframe restrictions.
